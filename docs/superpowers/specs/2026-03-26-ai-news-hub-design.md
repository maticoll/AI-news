# AI News Hub — Design Spec
**Date:** 2026-03-26
**Status:** Approved (v2 — added Gmail + categorization)

---

## Overview

A public web application for Spanish-speaking audiences that aggregates and displays news about new features from Claude and other major AI providers. Articles are scraped automatically from official blogs and email newsletters, summarized and categorized in Spanish using the Claude API, and presented in a clean, themeable interface with filtering by source and topic category.

---

## Goals

- Aggregate AI news from web sources (every 2 hours) and Gmail newsletters (every 2 hours)
- Generate short Spanish-language AI summaries using Claude Haiku
- Automatically categorize each article using a user-configurable category system
- Allow users to filter by source and by category; search across articles
- Support 3 visual themes (Clean Light default, Dark Tech, Gradient Modern) switchable in-page
- Public site — no login required

---

## Architecture

**Stack:** Python (FastAPI) + Vanilla JS frontend + SQLite

```
[APScheduler — every 2h, max_instances=1]
        │
        ├──→ [scraper.py]
        │       ├──→ blog.anthropic.com   (RSS via feedparser)
        │       ├──→ openai.com/news      (Playwright — JS-rendered)
        │       ├──→ deepmind.google      (Playwright — JS-rendered)
        │       └──→ mistral.ai/news      (Playwright — JS-rendered)
        │
        ├──→ [gmail_reader.py]
        │       └──→ Gmail API (reads newsletters from configured senders)
        │
        └──→ [summarizer.py]
                └──→ Claude API (claude-haiku-4-5)
                        ├── Spanish 2-sentence summary
                        └── Category assignment from categories.json

[SQLite DB] via SQLAlchemy
        │
[FastAPI] REST API
        │
[Frontend: index.html + app.js + themes.css]
```

## Database Schema

**Table: `articles`**

| Column | Type | Notes |
|---|---|---|
| id | INTEGER PK | Auto-increment |
| title | TEXT | Article title |
| url | TEXT UNIQUE | Used for deduplication |
| source | TEXT | e.g. "Anthropic", "OpenAI", "Newsletter" |
| source_type | TEXT | `"web"` or `"email"` |
| published_at | DATETIME | From RSS pubDate / HTML meta / email date; fallback: `scraped_at` (UTC ISO 8601) |
| ai_summary | TEXT | Generated by Claude Haiku (Spanish) |
| category_id | TEXT | Category `id` from `categories.json`; null until processed |
| scraped_at | DATETIME | When scraped (UTC ISO 8601) |
| is_processed | BOOLEAN | `false` on insert; `true` after successful AI summary + categorization |
| retry_count | INTEGER | Starts at 0; incremented on each failed Claude API attempt; max 5 |

**`is_processed` semantics:** Set to `false` on initial insert. The summarizer queries `WHERE is_processed = false AND retry_count < 5`. On successful Claude call: sets `is_processed = true`, `category_id`, and `ai_summary`. On failure: increments `retry_count`, keeps `is_processed = false`. After 5 failures, article is displayed without summary or category.

## API Endpoints

| Method | Path | Description |
|---|---|---|
| GET | `/api/articles` | Paginated list. Params: `source`, `category`, `q` (search), `limit` (default 20), `offset` |
| GET | `/api/sources` | Returns `["Anthropic", "OpenAI", "Google DeepMind", "Mistral", "Newsletter"]` |
| GET | `/api/categories` | Returns list of category objects from `categories.json` |
| POST | `/api/refresh` | Trigger manual scrape. Requires `X-Refresh-Token` header. Returns 401 if invalid. |

FastAPI serves the frontend's static files directly.

**Rate limiting (slowapi):** 60 req/min per IP (all endpoints); 5 req/min per IP (`/api/refresh`).

## Web Scraping (`scraper.py`)

- **Anthropic** (`blog.anthropic.com/rss.xml`): `httpx` + `feedparser`. Provides title, url, published_at.
- **OpenAI** (`openai.com/news`): Playwright headless Chromium.
- **Google DeepMind** (`deepmind.google/research/publications`): Playwright headless Chromium.
- **Mistral** (`mistral.ai/news`): Playwright headless Chromium.

Each extractor returns `[{title, url, published_at, excerpt}]`. BeautifulSoup4 used to extract plain-text excerpt from Playwright's rendered HTML output (first 500 chars of body text). `published_at` falls back to current UTC if not parseable. Deduplication by URL. Per-source failure: log and continue.

## Gmail Newsletter Reader (`gmail_reader.py`)

- Uses **Gmail API** (OAuth 2.0). Credentials JSON path stored in `GMAIL_CREDENTIALS_PATH` env var.
- OAuth 2.0 token must be **pre-generated on a local machine** (run `python gmail_reader.py --auth` once, which opens a browser for consent and saves `gmail_token.json`). Deploy `gmail_token.json` alongside the app. The server never opens a browser — it only uses the cached token and refreshes it automatically when it expires.
- **Sender filter:** reads `NEWSLETTER_SENDERS` from `.env` — comma-separated list of email addresses (e.g. `tldr@tldr.tech,newsletter@aiweekly.com`).
- Each cycle: fetches unread emails from those senders received since last check. Marks as read after processing.
- Extracts: subject (→ title), sender name (→ source), date (→ published_at), HTML body stripped to plain text (→ excerpt, first 500 chars).
- `source_type = "email"`, `source = sender display name`.
- Deduplication: uses `Message-ID` header as the unique `url` field value. If `Message-ID` is absent or empty, fallback key is `sha256(sender_email + subject + date_string)`. This synthetic key is used as the `url` column value.

## AI Processing (`summarizer.py`)

Single Claude Haiku call per article that produces both summary and category:

**Prompt structure:**
```
Given this AI news article, respond with JSON only:
Title: {title}
Excerpt: {excerpt}

Available categories: {categories_json}

Respond with:
{
  "summary": "<2-sentence summary in Spanish>",
  "category_id": "<best matching category id>"
}
```

- Reads `categories.json` at the start of each summarizer job run (no file-watching needed — always fresh on each cycle)
- On successful response: sets `ai_summary`, `category_id`, `is_processed = true`
- On API error or invalid JSON response: increments `retry_count`, keeps `is_processed = false`

## Scheduler (`scheduler.py`)

- `APScheduler` AsyncIOScheduler, every 2 hours, `max_instances=1`
- Job order: `scrape_all()` → `read_gmail()` → `summarize_pending()`
- Starts on FastAPI lifespan startup event

## Frontend (`app.js`)

- On load: fetches `/api/articles?limit=20&offset=0` + `/api/categories` + `/api/sources`
- **Dual filter bar:** source chips row + category chips row (both independent, combinable)
- **Pagination:** "Cargar más" button, 20 per page, offset-based, resets on filter/search change
- Search: debounced 300ms, `?q=` param, resets offset
- Theme switcher: `data-theme` on `<body>`, `localStorage["ai-news-theme"]`, default `"light"` on first visit
- Cards display: title, source badge, category badge, relative time, AI summary (placeholder "Procesando..." if pending), "Leer más" link
- On fetch error: inline Spanish error message

## Error Handling

- Web scraping failure per source: log + continue
- Gmail API auth failure: log, skip email step, continue with web scraping
- Claude API failure: increment `retry_count`, keep `is_processed = false` (max 5 retries)
- Invalid `category_id` in Claude response: fall back to `"otros"`
- API rate limiting: slowapi per-IP
- Unauthenticated `/api/refresh`: 401
- Frontend fetch errors: inline Spanish message
